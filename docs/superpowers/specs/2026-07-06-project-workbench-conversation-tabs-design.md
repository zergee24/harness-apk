# Project Workbench And Conversation Grouping Design

## Goal

项目模块从“Markdown 文件编辑页”调整为“项目工作台”。项目是长期工作的容器，会话是主要工作入口，文件夹是会话沉淀后的产物视图。

## Information Architecture

项目页分为两层：

1. 项目核心信息
   - 展示当前项目名称和简要状态。
   - 支持切换项目。
   - 支持新建项目。
   - 支持新建当前项目会话。
   - 后续可补充项目上下文入口，但不要把文档分类和命名作为主路径。

2. 项目内容二级 tab
   - 默认 tab：会话列表。
   - 第二个 tab：文件夹。
   - tab 样式应明显弱于底部一级导航，表达它属于当前项目内部。

## Project Conversations Tab

会话 tab 是项目页默认状态。

- 只展示当前项目关联的会话。
- 会话按更新时间倒序排列。
- 每个会话项展示标题、最近更新时间、模型或状态信息。
- 支持进入会话继续工作。
- 支持新建当前项目会话。
- 项目会话本质仍是普通会话，只是带有 `projectId` 元数据；不要引入另一套会话系统。

## Project Folder Tab

文件夹 tab 展示项目本地沉淀文件。

- 默认展示当前项目下的 Markdown 文件。
- 保留搜索、查看、预览、必要时编辑能力。
- 文件夹视图不是主要增删改入口；主要增删改仍由会话中的“沉淀到项目”和 diff 审核驱动。
- 后续 ppt、excel、pdf、code/git 交付物都应进入这一层，而不是提升为项目页主路径。

## Top-Level Conversation Mode

一级“会话”模式保持最近会话流优先，避免牺牲快速续聊效率。

- 默认仍按最近更新时间展示所有普通会话和项目会话。
- 项目会话在列表中显示项目标签。
- 提供“按项目分组”的切换能力。
- 分组视图中：
  - 非项目会话归入“普通会话”。
  - 项目会话按项目分组。
  - 项目组按组内最近更新时间排序。
  - 组内会话按更新时间排序。
  - 项目组 header 支持进入对应项目页。

## Data Model

会话需要持久化项目归属。

- `Conversation` 增加可空 `projectId`。
- 新建项目会话时写入 `projectId`。
- 普通会话 `projectId` 为空。
- ChatScreen 进入时如果路由携带 `projectId`，需要把会话归属持久化到当前 conversation。
- 一级会话页和项目会话 tab 都基于持久化 `projectId` 过滤或分组，不使用标题前缀、文件路径或临时路由参数推断。

## Migration And Backfill

数据库迁移新增可空 `projectId` 字段。

- 新会话可以可靠记录项目归属。
- 旧的项目会话如果没有结构化归属，不强行通过标题回填。
- 旧会话仍保留在普通会话流里。
- 如后续需要回填，可单独设计“移动到项目”能力。

## Non-Goals

- 本轮不做跨项目会话移动。
- 本轮不做项目会话归档策略重构。
- 本轮不做复杂文件树和多类型交付物管理，只保留当前 Markdown 文件视图。
- 本轮不改变“沉淀到项目”的 diff 审核规则。

## Acceptance Criteria

- 进入项目页后，顶部首先看到项目核心信息和切换能力。
- 当前项目下默认展示会话列表，不默认进入 Markdown 编辑器。
- 项目内部可以在“会话”和“文件夹”二级 tab 间切换。
- 新建项目会话后，该会话能在项目会话 tab 中稳定出现。
- 一级会话页默认仍是最近会话流，项目会话带项目标签。
- 一级会话页支持切换到按项目分组展示。
- 文件夹 tab 继续复用共享 Markdown 渲染组件。
