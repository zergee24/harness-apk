# 可移植语料人格智能体设计

## 1. 产品目标

Harness APK 新增第三种顶层模式“智能体”。用户在桌面 Codex 中自行导入有权使用的书籍和文本资料，构建一个可版本化、可验证、可移植的第一人称模拟智能体，再通过本地文件、局域网或远端智能体源安装到 Harness APK 中使用。

智能体不是对真实人物的复刻声明，而是“基于用户所选资料构建的模拟代理”。它使用第一人称表达，但核心判断必须能回到已安装资料；资料不足时明确说明不能据此判断，不使用模型通用知识补齐人物立场。

## 2. 第一性原则

1. `Agent` 是独立于会话和项目的可复用资产，拥有自己的语料、版本和质量状态。
2. 桌面 Codex 负责重型构建，Harness APK 负责安装、检索、对话和引用展示；不依赖常驻服务端。
3. 人格定义、标准化全文和原始文件分层打包，避免每次人格更新都重新传输全部书籍。
4. 原文证据与人格表达分离。没有已安装证据的回答不能伪装成资料结论。
5. 远端分发采用开放的 HTTPS catalog 协议，不把智能体列表写死在 APK，也不绑定单一 OSS。
6. 本地文件导入必须完整可用；远端下架或网络失败不能使已安装智能体失效。
7. 任何包都只包含声明式资料，不允许脚本、插件或可执行工具配置。

## 3. 信息架构

首页顶层模式调整为：

```text
会话 | 项目 | 智能体
```

三者职责：

| 模式 | 核心实体 | 职责 |
| --- | --- | --- |
| 会话 | `Conversation` | 执行一次临时或持续交流 |
| 项目 | `Project` | 围绕文件、项目会话和 Git 推进工作 |
| 智能体 | `Agent` | 构建结果的安装、资料覆盖、版本和启动入口 |

智能体模式不是第三套聊天实现。用户从智能体详情点击“开始对话”后，仍创建普通 `Conversation`，只是会话额外固定保存 `agentId + agentVersion`。会话还可以同时带有 `projectId`，从而让智能体参与项目工作。

第一期不建立 `Project <-> Agent` 直接关联表。是否在一个项目中使用某智能体，由同时包含 `projectId` 和 `agentId` 的会话表达，避免增加重复关系。

## 4. 双端总体架构

```text
桌面 Codex
书籍 -> 提取 -> 分块 -> 分层蒸馏 -> 评估 -> 签名 -> Agent 包
                                                        |
                 本地文件 / 局域网 / HTTPS catalog -----+
                                                        |
Harness APK                                             v
校验 -> 安装 -> 本地全文索引 -> 检索 -> Provider 回答 -> 来源验证
```

桌面 Codex 不是常驻 API。构建和更新智能体是用户主动发起的 Codex 任务；确定性处理由仓库内构建器完成，语义蒸馏由 Codex 分批执行。Harness 不等待桌面在线，也不要求两端账户同步。

## 5. 桌面智能体构建器

仓库提供可由桌面 Codex 调用的 `agent-builder` 工具。构建过程分为四个明确阶段：

1. `prepare`：提取文本、识别章节、生成稳定来源 ID 和分块任务。
2. `distill`：Codex 按章、按书、跨书分层生成结构化结果。
3. `validate`：执行引用、覆盖率、检索和包结构检查，输出构建报告。
4. `pack`：生成版本化包、校验清单和签名，不重新执行语义蒸馏。

第一期输入格式：

- TXT。
- Markdown。
- EPUB。
- 有文本层的 PDF。

扫描 PDF 和图片 OCR 不在第一期范围。提取失败的文件不能静默跳过；构建报告必须列出失败文件，发布质量门槛要求声明的输入文件全部成功处理。

### 5.1 分层蒸馏

蒸馏流程：

```text
原文块分析
-> 单章观点与证据
-> 单书概念和论证结构
-> 跨书思想框架
-> 时间变化与矛盾保留
-> Persona、示例和评估集
```

每条蒸馏观点都保存结构化证据：

```json
{
  "id": "view-method-001",
  "statement": "调查应先于结论形成",
  "evidence": ["chunk-0182", "chunk-0917"],
  "scope": "方法论",
  "period": "早期至中期",
  "confidence": 0.93
}
```

单一来源观点必须显式标记。不同书籍或时期存在冲突时，保留多个带适用时期和证据的观点，不强行合并成一个永恒人格。

### 5.2 构建产物

语义产物至少包括：

- `persona.md`：身份、第一人称表达方式、模拟标识和行为边界。
- `worldview.jsonl`：观点、概念关系、时期、置信度和证据 ID。
- `concepts.json`：概念词典、同义词和检索关键词。
- `examples.jsonl`：来自原文依据的典型表达与推理样例。
- `eval.jsonl`：代表性问题、期望覆盖点和必须命中的来源。
- `build-report.md`：覆盖率、冲突观点、失败问题和示例回答。

## 6. 四种包格式

### 6.1 `.hagent` 智能体核心包

包含人格、思想框架、示例、评估集、资料依赖和版本信息，不包含完整书籍正文。核心结构：

```text
manifest.json
persona.md
worldview.jsonl
concepts.json
examples.jsonl
eval.jsonl
checksums.json
signature.json
```

### 6.2 `.hcorpus` 标准化全文资料包

按书籍或卷册拆分，包含完整标准化文本而不是少量摘录：

```text
manifest.json
sources.json
chunks.jsonl
checksums.json
signature.json
```

每个 chunk 至少包含：

- `chunkId`。
- `sourceId`。
- 书名、版次、章节和可读位置。
- 标准化全文片段。
- 关键词、概念和中文 n-gram 检索字段。
- 原始来源内容哈希。

资料包按 `corpusId + sourceHash` 去重，可被多个 Agent 版本和多个智能体共享。

### 6.3 `.hsource` 原始附件包

包含 PDF、EPUB 或其他原始载体及其定位映射，只用于打开原始书页、阅读全文和人工核对，不参与常规检索。它完全可选，默认不随本地快速安装包加入。

### 6.4 `.hbundle` 本地便携安装包

`.hbundle` 是本地搬运容器，可以同时包含一个 `.hagent`、用户选中的多个 `.hcorpus` 和可选 `.hsource`。APK 导入后仍将内部资产独立安装、校验和去重。

远端更新不使用整包 `.hbundle`，避免只修改人格时重新下载全部语料。

## 7. 资料依赖与可用状态

`.hagent` manifest 声明资料依赖：

```json
{
  "corpora": [
    {
      "id": "selected-works-v1",
      "title": "第一卷",
      "sourceHash": "...",
      "required": true
    },
    {
      "id": "collected-works-v5",
      "title": "第五卷",
      "sourceHash": "...",
      "required": false
    }
  ]
}
```

Agent 状态：

- `WAITING_FOR_CORPUS`：缺少任一必需资料包，可以查看详情但不能开始对话。
- `READY`：必需资料已安装，可以严格基于当前资料回答。
- `DRAFT`：构建质量门槛未通过，只允许本地预览，不能发布到 catalog。
- `INVALID`：签名、结构或兼容性失败，不进入可用列表。

可选资料缺失不会阻止对话，但详情和来源面板显示当前资料覆盖，例如“已安装 3/8 份资料”。回答只能使用已安装资料；不能根据核心包的蒸馏结论伪造未安装原文引用。

不提供“完全没有 `.hcorpus` 仍正常聊天”的人格提示词降级模式。

## 8. Harness 智能体模式

### 8.1 列表

智能体模式包含两个紧凑视图：

- “已安装”：按最近使用排序，显示名称、当前版本、资料覆盖和状态。
- “发现”：聚合已启用远端源中的可安装和可更新智能体。

顶部只保留刷新和添加图标。没有市场化推荐流、点赞、评论或榜单。

### 8.2 详情

详情展示：

- “基于资料模拟”身份标识。
- 当前激活版本和可用更新。
- 已安装、缺失及可选资料包。
- 语料来源、版次和构建报告摘要。
- 版本说明与历史版本占用。
- 主操作“开始对话”。

### 8.3 会话启动

创建 Agent 会话时固定写入：

```text
conversationId
agentId
agentVersion
projectId?  // 可选
```

Agent 更新后，新会话默认使用最新激活版本；历史会话继续使用创建时版本。用户可以显式把旧会话迁移到新版本，但系统不自动改变进行中的人格和资料边界。

## 9. 移动端检索与回答

第一期不引入向量数据库、远程 RAG 服务或手机本地 embedding 模型。

APK 安装 `.hcorpus` 后，根据桌面生成的关键词、概念和中文 n-gram 建立 Room/SQLite FTS4 兼容索引。运行时：

1. 对用户问题执行本地全文和关键词检索。
2. 合并概念、来源与章节命中，取候选结果。
3. 去重后选择最多 8 个证据片段进入本轮模型上下文。
4. 注入固定 Agent 版本的 persona、适用 worldview 和示例。
5. 要求模型以第一人称回答，并在核心判断后使用本轮来源编号，例如 `[S1]`。
6. 流式完成后校验来源编号只能引用本轮真实提供的片段。

资料片段始终作为不可信引用内容注入，不能覆盖系统约束、开启工具、修改应用设置或改变 Agent 身份。

回答底部默认只显示“依据 N 处材料”。展开后展示原文片段、书名、版次、章节和位置；已安装 `.hsource` 时额外提供“打开原文件”。

检索没有足够证据时，第一人称回答应明确表达“根据当前已安装材料无法判断”。不调用通用知识补齐人物立场。

若模型生成不存在的来源 ID，回答标记为“引用未完全验证”，不伪造来源，并提供重试入口。

## 10. 本地导入体验

### 10.1 系统文件导入

智能体模式添加菜单提供“从文件选择”。使用 Android Storage Access Framework：

- 导入单个 `.hbundle`。
- 一次多选 `.hagent / .hcorpus / .hsource`。
- 支持从系统文件、聊天软件、网盘或邮件通过“用 Harness 打开”进入导入。
- 不申请整个外部存储权限。

选择后只出现一次安装确认：

```text
思想研究代理 v3

核心包             2 MB       必装
第一卷资料包       8 MB       已选择
第二卷资料包      11 MB       已选择
原始 PDF         186 MB       未选择

安装后占用约 34 MB
```

原始附件默认不选。导入过程流式复制、解压和校验，不把大文件一次性读入内存。

### 10.2 分享与打开

应用声明自定义 MIME 类型，并兼容文件管理器常见的 `application/zip` 和 `application/octet-stream`。通过 `ACTION_VIEW` 或 `ACTION_SEND` 接收后，仍进入同一安装预览和校验流程，不允许外部 Intent 绕过确认。

### 10.3 局域网扫码直传

局域网直传属于实施包 3：

1. 桌面构建器启动仅在本次任务期间可用的临时传输服务。
2. 生成一次性二维码，包含短期地址、令牌、包哈希和会话密钥。
3. Harness 扫码后展示与文件导入相同的安装预览。
4. 用户选择资料与原始文件，完成下载和安装。
5. 令牌成功使用一次或十分钟后失效，桌面服务自动关闭。

传输内容使用会话密钥加密，不上传到任何云端。

### 10.4 重复和升级

- 同版本、同哈希：提示已安装，不重复占用空间。
- 同版本、不同哈希：拒绝覆盖，要求构建新版本。
- 新版本：安装后由用户确认是否激活，保留历史会话引用的旧版本。
- 相同资料：按 `sourceHash` 跨智能体复用。
- 导入中断：只删除 staging 内容，不影响已安装版本。

## 11. 远端智能体源

### 11.1 通用协议

远端源是可配置的 HTTPS catalog，而不是写死的 Agent 列表。应用可以内置一个可编辑的默认源 URL，并允许用户添加自定义源。

目录结构：

```text
agent-catalog.json
agents/<agentId>/<version>/agent.hagent
corpora/<corpusId>/<sourceHash>/corpus.hcorpus
sources/<sourceId>/<sourceHash>/source.hsource
```

catalog 条目至少包含：

```json
{
  "id": "thought-study-agent",
  "name": "思想研究代理",
  "version": 3,
  "minAppVersionCode": 120,
  "packageUrls": ["https://example.com/agents/.../agent.hagent"],
  "sha256": "...",
  "signature": "...",
  "sizeBytes": 18392042,
  "releaseNotes": ["补充第五卷资料"],
  "publishedAt": "2026-07-16T12:00:00Z"
}
```

多个 `packageUrls` 只允许指向同一签名和 SHA-256 的不可变包。镜像用于可用性，不自动信任不同内容。

### 11.2 下载与激活

复用 APK 更新已经验证的网络模式：HTTPS、有限重试、分片、SHA-256、`.part` 临时文件、原子替换和 manifest 最后发布原则。Agent 下载使用独立 Repository、缓存目录、状态和错误文案，不复用 APK 安装器或 APK 下载状态。

可以自动检查和后台下载，但不能静默激活。激活前展示版本说明、资料变化和兼容性。安装失败继续使用当前版本。

### 11.3 来源信任

- 官方默认源固定发布公钥，不固定 Agent 列表。
- 自定义源首次添加时展示域名和公钥指纹，由用户确认信任。
- catalog 和包都必须签名；远端更新必须沿用已信任发布者密钥。
- 发布者密钥变化需要重新确认，不能随 catalog 静默轮换。

## 12. OSS 与内容治理边界

远端分发目标是降低单点依赖，不是绕过存储平台审核。

- 官方源只发布有明确授权、适合公开分发的 Agent 和资料包。
- 用户自行导入的完整书籍默认只存在于桌面、本地 `.hbundle` 和手机，不自动上传 OSS。
- 对存在版权或内容风险的资料，远端可以只发布构建配方和核心定义；全文通过本地文件或局域网导入。
- 改名、分片或加密不能作为规避平台规则的机制。
- 客户端加密只用于私人资料的隐私保护，仍必须遵守存储服务政策。

在没有服务端的情况下，私有 OSS 长期自动更新不是默认方案：私有对象需要动态签名 URL，而 APK 不能内置长期 AccessKey。桌面可生成短期 URL 用于临时传输；长期私人同步由实施包 3 的子规格明确密钥管理、恢复和撤销机制，本总规格不预设实现细节。

Harness 必须保留：

- 本地文件导入。
- 已安装版本离线运行。
- 多镜像但同签名包的下载回退。
- 远端不可用时不删除本地 Agent。

## 13. 安装安全

所有包先进入 staging，完整验证后再原子移动到版本目录。

验证规则：

- 只允许 manifest 声明的 JSON、JSONL、Markdown 和只读原始素材。
- 拒绝绝对路径、`..` 路径逃逸、符号链接和未声明条目。
- 拒绝脚本、动态库、DEX、APK、HTML/JS 插件和工具执行配置。
- 验证 schema、`minAppVersionCode`、发布者签名、每个文件 SHA-256 和总体积。
- 压缩包上限 2 GiB，声明的解压总量上限 4 GiB，条目上限 50,000；同时检查设备剩余空间。
- 解压失败、空间不足或校验失败只清理 staging，不影响已安装版本。

桌面构建器为每个发布者生成 Ed25519 密钥。私钥不进入包、不提交 Git；移动端只保存发布者公钥和指纹。即使本地个人包也应自签名，以便后续版本建立连续信任。

大正文、原始文件和索引文件保存在文件系统，Room 只保存元数据和关系，不复制语料正文。

## 15. 质量门槛

构建标记为可发布前必须满足：

1. 声明的输入文件全部提取成功。
2. 每条 worldview 观点引用的 chunk 均存在且来源哈希一致。
3. 单一来源观点已标记，不伪装为跨书共识。
4. 发布评估集不少于 `max(20, required corpus 数量 x 2)` 题，且每个必需资料包至少被两题覆盖。
5. 评估集检索 Top 8 的来源命中率不低于 85%。
6. 评估回答中每个可验证的观点句至少关联一个实际存在的 source ID；出现伪造来源、来源与观点明确矛盾或用通用知识补写人物立场时直接失败。
7. 自动证据一致性复核的观点支持率不低于 95%；未通过的观点必须人工修订或删除，不能通过放宽阈值发布。
8. 引用 ID、包内路径、checksums 和签名全部通过确定性验证。
9. 构建报告记录模型、构建器版本、输入来源哈希、逐题结果和构建时间。

未通过门槛的构建只能生成 `DRAFT` 本地预览包，发布工具拒绝写入远端 catalog。

## 16. 错误与恢复

| 场景 | 行为 |
| --- | --- |
| 文本提取失败 | 构建失败并列出具体文件，不生成发布包 |
| 缺少必需 `.hcorpus` | Agent 进入 `WAITING_FOR_CORPUS`，保留补装入口 |
| 包签名或 checksum 失败 | 拒绝安装，当前版本不变 |
| 同版本内容不同 | 拒绝覆盖，要求发布新版本 |
| FTS 索引中断 | 保留包但不标记 `READY`，允许重建索引 |
| catalog 不可用 | 展示源错误，已安装 Agent 继续离线使用 |
| 下载中断 | 删除 `.part`，允许重试，不破坏已安装版本 |
| 模型引用未知来源 | 标记“引用未完全验证”，不伪造来源，允许重试 |
| 当前资料无答案 | 明确材料不足，不调用通用知识补齐人物立场 |

## 17. 测试策略

### 桌面构建器

- TXT、Markdown、EPUB 和文本 PDF 的提取与稳定来源 ID。
- 失败输入不被静默忽略。
- 分块结果可重复，来源位置和哈希稳定。
- 评估题量或资料覆盖不足时验证失败。
- 引用不存在、观点与证据矛盾、观点支持率不足或检索命中率不足时验证失败。
- 相同输入和语义产物可重复打出相同 checksums。
- 私钥不进入包和日志。

### 包与安装器

- 四种包的 schema、checksum、签名和依赖解析。
- 路径逃逸、符号链接、zip bomb、超额条目和可执行文件拒绝。
- staging 失败不破坏已安装版本。
- bundle 拆分安装、资料去重和同版本冲突。
- 大文件流式导入，不出现明显内存峰值。

### 智能体运行时

- FTS 索引建立、重建、资料安装和移除。
- 只检索当前 Agent 版本声明且已安装的资料。
- Top 8 证据选择稳定，未知引用可被识别。
- 无证据问题进入材料不足回答。
- 普通会话不注入 Agent 上下文。
- 历史会话固定旧版本，新会话使用最新激活版本。

### 远端源

- catalog 签名、公钥指纹确认和密钥变化阻断。
- 多镜像只接受同 SHA-256 和签名内容。
- manifest 最后发布和不可变版本路径。
- 下载重试、`.part`、原子安装和失败保留旧版本。
- 远端删除 Agent 不删除本地已安装资产。

### UI 与 Android 集成

- 第三种顶层模式在手机宽度下不溢出。
- 文件选择、多选、分享打开和一次安装预览。
- 缺失资料、安装中、可用、草稿和失败状态可区分。
- 来源折叠行和原文详情可访问。
- APK 更新、项目、普通会话和现有文件导入不回归。

## 18. 验收标准

1. 用户能在桌面 Codex 以一批文本书籍生成通过校验的 `.hbundle`。
2. 用户在 Harness 中只选择一次文件即可安装核心和所选资料包。
3. 至少一个必需资料包安装完成后，Agent 可以创建固定版本会话。
4. 回答使用第一人称，核心判断可展开查看真实来源片段。
5. 未安装资料覆盖不到的问题不会被通用知识补齐为人物立场。
6. 同一 `.hcorpus` 可被两个 Agent 复用且只存储一次。
7. Agent 新版本安装后不改变历史会话，新会话默认使用新激活版本。
8. 远端 catalog 可以新增 Agent 和版本而不发布新版 APK。
9. 远端源不可用或对象下架后，已安装 Agent 仍可离线使用。
10. 包校验、安装或索引失败不会破坏现有可用版本。

## 19. 分阶段交付

### 实施包 1：本地构建与运行闭环

- 四种包格式和签名规范。
- 桌面 `prepare / validate / pack` 构建器骨架及 Codex 分层蒸馏约定。
- 第三种顶层模式、Agent/版本/资料数据模型。
- `.hbundle` 文件选择、分享打开、安装预览和原子安装。
- 本地 FTS 检索、第一人称 Agent 会话和引用验证。

### 实施包 2：远端源与版本更新

- AgentSource 和签名 catalog。
- 发现、检查更新、多镜像下载和显式激活。
- 不可变发布目录、catalog 最后上传和发布脚本。

### 实施包 3：便捷直传与私人同步

- 桌面临时传输服务和 APK 二维码扫描。
- 一次性令牌、会话加密和安装预览复用。
- 客户端加密的用户自有远端源方案；不在 APK 保存长期 OSS AccessKey。

第一份实施计划只覆盖实施包 1。实施包 2 和 3 在前一阶段验收后分别编写子规格和计划。

## 20. 非目标

- 智能体市场、评论、点赞和公开社区。
- 内置或分发用户未授权的书籍。
- 扫描件 OCR。
- 手机本地 embedding 模型或完整向量数据库。
- 服务端账户、同步和跨设备状态。
- Agent 自动修改自身 persona 或 worldview。
- 远端版本静默激活。
- 在 Agent 包中执行代码、插件或任意工具。
