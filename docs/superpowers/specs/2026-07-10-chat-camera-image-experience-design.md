# 会话拍照与图片体验设计

**日期：** 2026-07-10

## 目标

让用户能够在会话输入框内拍摄一张照片或从相册选择一张图片；图片发送后在会话中直接显示缩略图，点击打开全屏预览，并可由用户主动保存到系统相册。

## 已确认的产品决策

- 每条消息当前只支持一张图片。
- 输入框使用一个统一的“添加图片”入口，打开操作面板后提供“拍照”和“从相册选择”。
- 拍照完成后直接进入当前待发送图片预览，不增加确认或重拍页。
- 拍照和发送不会自动保存到系统相册。
- 用户在会话图片全屏预览页点击“保存图片”后，才将原图保存到系统相册。
- 会话中需要渲染用户发送的图片，以及模型回复中可解析的图片内容。

## 交互设计

### 输入框

1. 未选择图片时，输入框右侧的加号按钮命名为“添加图片”。
2. 点击后弹出底部操作面板：
   - 拍照
   - 从相册选择
3. 点击拍照时按需申请相机权限。授权后打开系统相机；取消、权限拒绝或相机失败时保留已输入的文字，不创建附件。
4. 成功返回后显示已有的单图待发送预览，可移除后继续编辑文字。

### 会话消息与预览

1. 图片附件在消息气泡中显示缩略图，而不是“图片附件”文字元信息。
2. 点击缩略图打开沉浸式单图预览页，提供关闭按钮和“保存图片”操作。
3. 保存成功显示简短反馈；存储失败、来源不可访问或远程图片加载失败时显示可理解的错误，不影响消息文本。
4. 模型图片支持现有图片消息部件以及 Markdown 图片链接；网络图片加载后缓存为本地可用副本。

## 数据与实现边界

### 持久图片附件

- 相册 URI 和相机临时 URI 在发送前复制到应用专属持久目录，`ChatAttachment` 保存应用可控的本地 URI、MIME 类型和消息关联。
- 相机输出 URI 由 `FileProvider` 提供，增加受限的相机缓存路径；不将拍照结果自动写入系统媒体库。
- 保留当前 8 MB 图片上限和发送前压缩策略；相机图片默认使用 `image/jpeg`。

### 模型图片

- 数据 URI、应用本地 URI 和 HTTPS 图片链接统一转换为可显示的图片来源。
- HTTPS 图片首次渲染时下载到缓存；下载失败只影响图片部件，不能让整条消息渲染失败。
- 下载/保存始终使用原始可获得内容；不可获得时明确提示。

### Android 平台能力

- 新增 `CAMERA` 声明与运行时权限请求。
- 使用 `ActivityResultContracts.TakePicture` 与现有 `PickVisualMedia`，两者都最终回填同一张待发送图片状态。
- 保存图片使用 `MediaStore` 写入用户媒体库；不要求宽泛存储权限。

## 错误处理

- 相机权限拒绝：提示可从相册选择，不阻断文字发送。
- 相机取消：无提示噪音，保持输入内容。
- 图片复制、读取、压缩或保存失败：显示单次错误提示，保留可恢复的会话状态。
- 不支持图片输入的模型：沿用现有能力校验，发送前提示切换模型或移除图片。

## 测试与验收

- 单元测试：附件复制、URI/MIME 归一化、模型图片来源解析、保存失败和图片能力校验。
- Compose 测试：添加图片面板、消息缩略图、点击预览、保存动作与错误反馈。
- Android 设备测试：相册选择回填、相机权限拒绝、模拟相机结果、预览保存到媒体库。
- 手工验收：拍照、发送、重启后历史图片渲染、全屏预览、保存图片，以及模型图片加载失败的降级展示。

## 非目标

- 多图选择、连拍、编辑/裁剪、滤镜和图片标注。
- 拍照后自动保存到相册。
- 在本次功能中重构项目工作台顶部；该方向已确认保留现有标签页优先结构，后续单独设计。
