# 项目名称修改设计

## 目标

让用户在项目工作台修改当前项目的显示名称，同时保持项目目录、项目 ID、会话关联和 Git 仓库状态不变。

## 交互

在已选项目的工作台顶部“更多”菜单中增加“重命名项目”。点击后打开预填当前名称的弹窗：

- 输入会保留用户键入内容，保存时去除首尾空格。
- 空名称不可保存，并保持现有的“项目名称不能为空”校验语义。
- 保存成功后关闭弹窗，显示“已重命名项目：<名称>”，并即时刷新项目标题、项目切换列表与应用顶部当前项目名。
- 取消或点击弹窗外部不产生修改。

## 数据与兼容

`FileProjectRepository.renameProject(projectId, name)` 只写入项目目录中的 `.harness/name` 元数据并更新项目根目录修改时间。项目目录名继续作为稳定 ID，因此既有会话的 `projectId`、交付物路径和 Git 仓库都不受影响。

创建项目和旧项目没有 `.harness/name` 时继续使用 README 一级标题或目录名作为回退名称。重命名后该本地元数据在应用重启后仍可恢复。项目包导出仍不携带 `.harness` 本地元数据，避免把应用本地显示设置写入项目仓库或分享包。

## 错误处理

仓储层复用现有项目目录存在性校验；写入失败在项目页显示统一的用户可读错误。UI 只在仓储层成功返回后更新名称和状态提示。

## 验证

- 仓储测试：重命名后 ID 与根目录不变，重新列出项目可读到新名称，空名称被拒绝。
- UI 状态测试：名称修改后当前项目标题使用新值。
- 全量 JVM 单测、Debug APK 和 Android 测试 APK 构建通过。
